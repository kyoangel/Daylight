# Daylight 使用者流程與頁面流程圖（P0）

更新日期：2026-02-03

## 目標
- 讓使用者能在低負擔的情境下，完成「每日情緒記錄 + 小任務 + 安定支持」
- 在情緒低落時，能快速進入 SOS 或安撫流程

---

## 全局導航結構（P0）
- 底部導覽：日常 / 陪伴 / 日記 / 我
- 全域 SOS 入口：日常、陪伴頁右上角或浮動按鈕

---

## 主要使用者流程（User Flows）

### Flow A：首次使用與個人化
1. OB-01 歡迎
2. OB-02 基本設定
3. OB-03 情緒偏好
4. OB-04 安全提示
5. 進入 DA-01 日常首頁

**出口**
- 正常：進入日常使用
- 中斷：可暫存後下次繼續

---

### Flow B：每日心情打卡
1. DA-01 日常首頁
2. 調整心情滑桿 + 點選小任務
3. CTA：完成打卡
4. DA-02 小任務完成
5. 回到 DA-01

**出口**
- 觸發晚安反思：DA-03
- 進入日記：DI-01

---

### Flow C：晚安反思
1. DA-03 晚安反思
2. 輸入回答
3. CTA：保存
4. 回到 DA-01

---

### Flow D：陪伴模式
1. CO-01 模式選擇
2. CO-02 對話頁
3. 10 分鐘陪伴結束或使用者結束
4. 回到 CO-01 或 DA-01

**出口**
- 使用者觸發 SOS：進入 SO-01

---

### Flow E：日記輸入與回顧
1. DI-01 日記首頁
2. 選擇心情 + 文字輸入
3. CTA：保存
4. 回到 DI-01

**分支**
- 進入 DI-02 週回顧（入口可放在首頁）

---

### Flow F：SOS 求助
1. SO-01 SOS 首頁
2. 選擇：
   - 一鍵求助
   - 安撫流程（SO-02）
   - 危機資源
3. 完成後返回上一頁

---

### Flow G：個人設定
1. PR-01 個人設定
2. 修改暱稱 / 主題 / 語言 / 提醒
3. 保存並返回

---

## 頁面流程圖（文字版）

入口流程：
OB-01 -> OB-02 -> OB-03 -> OB-04 -> DA-01

主流程（每日）：
DA-01 -> DA-02 -> DA-01
DA-01 -> DA-03 -> DA-01
DA-01 -> DI-01 -> DI-02 -> DI-01
DA-01 -> CO-01 -> CO-02 -> CO-01

支援流程：
DA-01 / CO-02 / DI-01 -> SO-01 -> SO-02 -> 返回上一頁

設定流程：
DA-01 / CO-01 / DI-01 -> PR-01 -> 返回上一頁

---

## 風險節點與緩解
- Onboarding 可能中斷：支援暫存與略過
- 情緒低落時操作負擔高：提供一鍵 SOS
- 內容太多造成壓力：每頁最多 1-2 主 CTA
